# CSV to Excel 変換ツール 利用マニュアル

## 0. クイックガイド（これだけでもＯＫです🚀）

**「CSVファイルをドロップするだけ」で、文字化けさせずにExcelで開くツールです。**

### 😺 こんな時に便利です！
*   CSVを開いたら**文字化け**して読めない。
*   毎回「データからインポート」などの面倒な設定をしたくない。
*   **元のCSVファイルを絶対に書き換えたくない（壊したくない）。**

---

## ✅ 使い方
1.  **フォルダを保存**：`CSV_2_Excel` フォルダを、PCの任意の場所（デスクトップ等）にコピーします。
2.  **ショートカット作成**：フォルダ内の `【ここにCSVをドロップ】CSVをExcelで開く.bat` を右クリックし、デスクトップにショートカットを作成すると便利です。
3.  **ドロップして起動**：開きたいCSVファイルを、作成したショートカットに**ドラッグ＆ドロップ**するだけ！

---

## ⚠️ 重要な注意点
*   **「上書き保存」はできません**：
    本ツールはオリジナルのデータを保護するため、一時コピーを作成してExcelで開いています。編集内容を保存したい場合は、Excelの **「名前を付けて保存」** から、Excelブック（.xlsx）等として別名保存してください。
*   **フォルダ内ファイルはセットです**：
    バッチファイル（.bat）とスクリプト（.ps1）が同じフォルダにある必要があります。ファイル単体で移動させないでください。

# テクニカルリファレンス（補足資料）

## 1. 導入の背景と目的
Windows環境のExcelでCSVを扱う際、仕様上の制約により「文字化け」や「意図しないデータ書き換え」が発生し、業務効率低下のリスクとなっています。

### 1.1 文字化けおよびデータ破損の発生原因
1.  **Excelの文字コード自動判別仕様**
    *   日本版Excelは、CSVを開く際の既定値として「Shift_JIS」を使用します。
    *   現代の標準である「UTF-8（BOMなし）」形式のCSVをダブルクリックで開くと、ExcelはこれをShift_JISとして強引に読み込もうとし、文字化けが発生します。
2.  **BOM（バイトオーダーマーク）の欠如**
    *   BOMとは、ファイルの先頭に付与される「これはUTF-8です」という識別情報です。
    *   多くのWebシステムや最新ツールが出力するCSVにはBOMが含まれていないため、Excel側で識別ができず、誤った解釈が行われます。
3.  **データ破損のリスク**
    *   文字化けした状態で不用意に「保存」を行うと、文字化けした状態のままファイルが更新され、元の正しいデータへ復元することが困難になります。

### 1.2 現代のシステム環境における課題
*   **UTF-8の標準化**: Windows 10（1903以降）のメモ帳の既定値がUTF-8になり、Python等のプログラミング言語も標準でBOMなしUTF-8を出力します。
*   **Excelのローカライズ格差**: 中国語版Excelでは既にUTF-8がデフォルトの読み込み形式として採用されていますが、日本版Excelは依然として旧来のShift_JIS優先の挙動を維持しています。
*   **インポート工数**: 手動で回避するには、Excel起動後に「データ」タブから複雑なステップ（5クリック以上）を踏む必要があり、頻繁な作業には適しません。

---

## 2. 本ツールの解決策と動作原理

### 2.1 概要
本ツールは、CSVの文字コードを自動判定し、Excelが正しく認識できる「UTF-8 BOM付き」へ変換した上で起動します。

*   **効率化**: ドラッグ＆ドロップのみの1ステップで完了。
*   **安全性**: 元ファイルを一切変更せず、一時ファイル（%TEMP%）を介して開くため、オリジナルの破壊を物理的に防ぎます。

### 2.2 処理フロー
1.  **制御層（Batch）**: ファイルパスを受け取り、PowerShellエンジンへ橋渡しを行います。
2.  **処理層（PowerShell）**:
    *   元の文字コード（UTF-8 / Shift-JIS等）を自動判定。
    *   UTF-8（BOM付き）形式へ変換。
    *   一時ディレクトリ（%TEMP%）へ、重複しないファイル名で出力。
3.  **表示層（Excel）**: 生成された一時ファイルをExcelで展開。
4.  **自動消去**: Excel終了を検知後、一時ファイルを自動的に削除し、PC内をクリーンに保ちます。

---

## 3. 動作環境・要件
*   **OS**: Windows 10 / 11
*   **Excel**: Microsoft Excel 2016 以降推奨
*   **PowerShell**: バージョン 5.1（OS標準搭載）

## 4. トラブルシューティング
*   **スクリプトが動かない**:
    社内PC等の制限によりPowerShellの実行が禁止されている場合があります。本ツールは実行ポリシーを一時的にバイパス（`-ExecutionPolicy Bypass`）する設定を含んでいますが、システム全体の制約が優先される場合があります。
*   **特定の古い形式**:
    EUC-JPやJISコードなどの特殊な古い形式には対応していません。
