# CSV to Excel 変換ツール 利用マニュアル

## 0. クイックガイド（これだけでもＯＫです🚀）

本ツールは、CSVファイルをドラッグ&ドロップするだけで、文字コードの自動変換およびデータ型の最適化を行い、Excelで正確に表示するためのものです。

### 本ツールの機能
*   **文字化けの解消**: 文字コード（UTF-8/Shift_JIS）を自動判定し、適切に表示します。
*   **データ変質の防止**: Excelの仕様による「0落ち（先頭の0が消える）」や「指数表記（長い数字が省略される）」を防止し、元の値を保持したまま表示します。
*   **工程の短縮**: 通常のExcel操作（データタブからのインポート設定）を自動化し、1ステップで実行します。
---
## 1. 使用方法 STEP 1

1.  **フォルダの配置**
    `CSV_2_Excel` フォルダを、PC内の任意のディレクトリ（例：デスクトップ、ドキュメントなど）に配置します。
2.  **ショートカットの作成**
    フォルダ内の `【ここにCSVをドロップ】CSVをExcelで開く.bat` を右クリックし、**「その他のオプションを表示」** ＞ **「送る」** ＞ **「デスクトップ（ショートカットを作成）」** を選択します。
3.  **実行方法**
    表示したいCSVファイルを、作成したショートカットに **ドラッグ&ドロップ** します。自動的にExcelが起動し、内容が表示されます。

## 2. 使用方法 STEP 2  **【重要】数値データを認識させる**
    Excelが起動した直後では、数値データは文字列として認識しています（0落ちや指数形式を避けるためです）。次のように操作をして、数値データとして認識させてください。

1. 数値化したいセル範囲を選択
    例：A列全部など。

2. Excel 上部の **「データ」**タブ ＞ **「区切り位置」**をクリック

3. **「区切り位置指定ウィザード」**が展開されるが、何も触らずに **「完了」**を押す。
    **何もせずに「完了」だけを押す**のがポイントです。
---

## 3. 使用上の注意点

*   **「保存」の挙動について**：
    本ツールは、新規のExcelワークブックを作成し、そこにCSVのデータを「文字列型」として転送して表示します。CSVファイルを直接開いている状態ではないため、**「上書き保存」はできません。** 編集内容を保存する場合は、Excelの **「名前を付けて保存」** から、Excelブック（.xlsx）形式等を選択して保存してください。
*   **数値計算について**：
    「0落ち」等のデータ変質を防ぐため、すべてのセルを「テキスト（文字列型）」として展開します。そのため、そのままではSUM関数などの計算対象になりません。計算を行う必要がある列は、Excelの機能を使用して数値型へ変換してください。
*   **ファイル構成の維持**：
    実行用のバッチファイル（.bat）と処理用のスクリプト（.ps1）は、同一のフォルダ内に配置されている必要があります。単体では動作しませんので、ファイルを分離させないでください。
*   **処理時間**：
    プログラムを介してExcelを制御しデータを転送するため、行数の多いCSVファイル（数万行以上）では、展開完了までに数秒から十数秒程度の時間を要する場合があります。

# テクニカルリファレンス（興味があったら読んでください）

## 1. 導入の背景と目的
Windows環境のMicrosoft Excelにおいて、CSV（Comma Separated Values）ファイルを扱う際、仕様上の制約により「文字化け」や「数値データの意図しない自動変換」が発生します。これらの現象は、業務効率の低下を招くだけでなく、データの整合性を損なうリスクを伴います。
本ツールは、プログラムからExcelのインスタンス（実行状態）を直接制御することにより、文字コードの適合化とデータ型の保護を同時に行い、元の情報を正確に表示することを目的としています。

### 1.1 文字化けおよびデータ変質の発生原因
1.  **符号化方式（文字コード）の不一致**
    日本語版Excelは、CSVファイルを開く際の既定のエンコードとして「Shift_JIS（CP932）」を使用します。一方、現代の多くのシステムが出力する「UTF-8」形式のファイルをこの既定動作で処理すると、符号化方式の誤認により文字化けが発生します。
2.  **BOM（Byte Order Mark）の欠如**
    BOMとは、ファイルの先頭に付与される文字コード識別用のデータです。多くのWebシステムが出力するUTF-8形式にはBOMが含まれておらず、Excel側での自動判別を困難にしています。
3.  **データ型の自動推論による変質**
    Excelには、読み込んだ文字列を数値や日付と見なして自動変換する「型推論」機能が備わっています。これにより、以下のデータ変質が発生します。
    *   **先頭零の欠落（0落ち）**: 数字のみで構成される文字列の先頭の「0」が削除される現象。
    *   **指数表記への変換**: 12桁以上の数値が「1.23E+11」等の指数形式に短縮され、下位桁の情報が失われる現象。

### 1.2 現代のシステム環境における課題

1.  **UTF-8の標準化**  
    Windows 10（バージョン1903以降）のメモ帳、Python等の開発環境、各種クラウドサービスでは、UTF-8（BOMなし）が標準の文字コードとして採用されています。これにより、国際的に流通するCSVファイルの多くがUTF-8へ移行し、従来のShift_JISとの乖離が一層拡大しています。

2.  **日本語版Excelのローカライズ仕様**  
    海外版ExcelではUTF-8が既定の読み込み形式として広く採用されています。一方、日本語版ExcelではOSの地域設定（ロケール）に基づき、既存の長年使われてきた基幹システムが出力するShift_JISデータとの互換性を優先して「Shift_JIS」が既定値として維持されています。

3.  **レガシーシステムの影響**  
    国内の多くの基幹システムは、現在に至るまでShift_JIS形式でのデータ出力を前提として設計されています。仮にExcel側が既定仕様をUTF-8へ変更した場合、これら既存システムを利用する業務フローにおいて大規模な文字化けが発生する可能性が高く、Microsoft社としても安易に仕様変更できない状況にあります。

4.  **インポート手順の非効率性**  
    Excelの「データ」タブから外部データとして取り込む方法を用いれば、文字コードやデータ型を正しく指定して読み込むことは可能です。しかし、この方法は「データの取得」→「テキストまたはCSVから」→「プレビューでの型指定」など複数の操作を要するため、日常的なデータ確認作業においては効率を大きく損なう要因となっています。

---
## 2. 課題の技術的要因
### 2.1 文字コードの不一致
*   **既定の処理**: 日本語版Excelは、CSVファイルを開く際のエンコードとして「Shift_JIS（CP932）」を前提としています。
*   **標準の乖離**: 現代の多くのシステムが出力する「UTF-8」形式のファイルを既定で開くと、符号化方式の不一致により文字化けが発生します。

### 2.2 データ型の自動判定（型推論）
Excelには、読み込んだ文字列を解析し、自動的に最適なデータ型へ変換する機能が実装されています。この機能により、以下の現象が発生します。
*   **先頭零の欠落（0落ち）**: 数字のみで構成される文字列において、先頭の「0」が数学的に不要と判断され、除去される現象。
*   **指数表記への変換**: 12桁以上の数値が、指数形式（例：1.23E+11）に変換され、下位桁の情報が不可視化される現象。

## 3. 本ツールの解決策と動作原理
本ツールは、物理的なCSVファイルをExcelで直接開くのではなく、プログラム（PowerShell）がExcelアプリケーションを構成要素（COMオブジェクト）として制御し、データをメモリ上で転送する方式を採用しています。

### 3.1 処理フロー
1.  **データの読み取り**: 指定されたCSVファイルの文字コード（UTF-8 または Shift_JIS）を自動判定し、テキストデータとしてメモリ上に読み込みます。
2.  **Excelプロセスの制御**: Excelアプリケーションをバックグラウンドで起動し、新規ワークブックを作成します。
3.  **書式設定の強制**: データを入力する前に、対象のセル範囲に対して、表示形式を「文字列（@）」に指定します。
4.  **データの転送**: メモリ上のテキストデータをExcelのセルへ直接代入します。この際、セルが事前に「文字列型」として定義されているため、Excelによる自動型判定（数値変換）が抑制されます。
5.  **表示と最適化**: 全データの転送後、列幅の自動調整（AutoFit）を実行し、Excelウィンドウを前面に表示します。

### 3.2 本方式の利点
*   **非破壊性**: 元のCSVファイルは読み取り専用として扱うため、データの書き換えが発生しません。
*   **整合性の維持**: セルの表示形式を事前に固定することで、文字化け、0落ち、指数表記のすべてを構造的に防止します。

## 4. 仕様および制限事項
### 4.1 編集後の保存に関する挙動
本ツールで展開したデータは、CSVファイルを直接編集している状態ではなく、新規ワークブックへ内容を転写した状態となります。そのため、保存を実行する際は、Excelブック（.xlsx）形式での保存が既定となります。CSV形式で保存し直す場合は、再度Excelの出力仕様が適用される点に留意してください。

### 4.2 計算機能への影響
すべてのデータが「文字列型」として展開されるため、数値計算（SUM関数等）の対象外となります。計算が必要な列については、ユーザーがExcel上で適宜「数値型」へ書式変更を行う必要があります。
