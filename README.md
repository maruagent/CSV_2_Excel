# CSV → Excel 起動ツール（Drag & Drop）

CSVファイルをドラッグ＆ドロップするだけで、文字コードを自動判定し、文字化けなくExcelで開くことができるツールです。

## 概要

### 解決する課題

| 課題 | 解決策 |
|------|--------|
| CSVファイルをダブルクリックしてもExcelで正しく開かれない | 専用のバッチファイル経由で確実にExcelを起動 |
| 文字コード（Shift-JIS / UTF-8）問題による文字化け | 自動判定してUTF-8 BOMに変換 |
| Excel起動 → ファイル選択の操作コスト | ドラッグ＆ドロップのみで完了 |
| 非エンジニアがPowerShellやExcelインポート機能を扱えない | シンプルな操作で誰でも使用可能 |

### 対応文字コード

- ✅ UTF-8（BOMあり・なし両対応）
- ✅ UTF-16 LE
- ✅ Shift-JIS（CP932）

## インストール手順

### ステップ1: ファイルの準備

1. 本リポジトリをダウンロードまたはクローンします
2. 以下の2つのファイルが含まれていることを確認してください：
   - `【ここにCSVをドロップ】CSVをExcelで開く.bat`
   - `CsvToExcel_backend.ps1`

### ステップ2: フォルダの配置

任意の場所にフォルダを配置します。例：
```
C:\Tools\CSV_2_Excel\
├── 【ここにCSVをドロップ】CSVをExcelで開く.bat
└── CsvToExcel_backend.ps1
```

### ステップ3: ショートカットの作成（オプション）

デスクトップやタスクバーにショートカットを作成すると便利です：

1. `【ここにCSVをドロップ】CSVをExcelで開く.bat` を右クリック
2. 「ショートカットの作成」を選択
3. 作成されたショートカットをデスクトップなどに移動

## 使用方法

### 基本的な使い方

#### 方法1: バッチファイルに直接ドロップ

1. CSVファイルをエクスプローラーで表示
2. CSVファイルを `【ここにCSVをドロップ】CSVをExcelで開く.bat` のアイコンにドラッグ＆ドロップ
3. Excelが自動的に起動し、CSVが開きます

```
┌─────────────────────────────────────┐
│  📁 エクスプローラー                 │
│                                     │
│  📄 data.csv  ───────┐              │
│  📄 list.csv         │              │
│                      ▼              │
│  ┌─────────────────────────────┐    │
│  │ 🚀 CSVをExcelで開く.bat     │    │
│  │    【ここにドロップ】        │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

#### 方法2: ショートカットを使用

1. デスクトップなどに作成したショートカットを開いておく
2. CSVファイルをショートカットにドラッグ＆ドロップ

### 複数ファイルの一括処理

複数のCSVファイルを選択して一括でドロップすることも可能です（各ファイルが順番に処理されます）。

## 機能詳細

### 自動文字コード判定

本ツールは以下の順序で文字コードを自動判定します：

1. **UTF-8 BOM** - BOM（Byte Order Mark）がある場合
2. **UTF-16 LE** - Unicodeリトルエンディアン
3. **UTF-8（BOMなし）** - 有効なUTF-8として解析可能な場合
4. **Shift-JIS** - 上記以外の場合（日本語Windowsのデフォルト）

### 処理フロー

```
[CSVファイルをドロップ]
         ↓
[バッチファイルが起動]
         ↓
[PowerShellスクリプト実行]
         ↓
[文字コード自動判定]
         ↓
[UTF-8 BOMに変換]
         ↓
[一時ファイル作成（GUID付き）]
         ↓
[Excelで開く]
         ↓
[バックグラウンドで監視開始]
         ↓
[Excel終了を検知]
         ↓
[一時ファイルを自動削除]
```

### 一時ファイルについて

- 変換されたCSVは一時フォルダ（`%TEMP%`）に保存されます
- ファイル名にはGUID（ランダムな8文字）が付与され、同名ファイルの衝突を防止
- **自動削除機能**: Excelを閉じると、バックグラウンドで一時ファイルが自動的に削除されます（手動で削除する必要はありません）

## トラブルシューティング

### よくある問題と解決策

#### ❌ 「ファイルが見つかりません」エラー

**症状**: エラーメッセージ「ファイルが見つかりません。対象のCSVをバッチファイルにドロップしてください。」が表示される

**原因と解決策**:
- CSVファイルが存在しない、またはパスが無効
  - → ファイルが存在することを確認
- ネットワークドライブ上のファイル
  - → ローカルにコピーしてから試行

#### ❌ Excelが起動しない

**症状**: 処理は成功するがExcelが開かない

**原因と解決策**:
- Excelがインストールされていない
  - → Microsoft Excelのインストールが必要
- Excelの関連付けが破損
  - → Excelの修復インストールを実行

#### ❌ PowerShell実行ポリシーのエラー

**症状**: 「このシステムではスクリプトの実行が無効になっています」などのエラー

**解決策**:
PowerShellを管理者として実行し、以下のコマンドを実行：
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
または、バッチファイル内で `-ExecutionPolicy Bypass` を使用しているため、通常は発生しません。

#### ❌ 文字化けが発生する

**症状**: Excelで開いたCSVの日本語が文字化けする

**原因と解決策**:
- 特殊な文字コード（EUC-JPなど）を使用している
  - → 事前にUTF-8またはShift-JISに変換
- バイナリデータが含まれている
  - → CSVファイルの内容を確認

#### ❌ 「PowerShell script failed」エラー

**症状**: バッチファイルで「[ERROR] PowerShell script failed」と表示される

**確認事項**:
1. `CsvToExcel_backend.ps1` が同じフォルダに存在するか確認
2. PowerShellが正常に動作するか確認：
   ```cmd
   powershell -Command "Get-Host"
   ```

### サポート情報

問題が解決しない場合は、以下の情報を収集してください：

1. Windowsのバージョン
2. Excelのバージョン
3. エラーメッセージの全文
4. 問題が発生するCSVファイルの文字コード（可能であれば）

## 技術仕様

### ファイル構成

```
CSV_2_Excel/
├── 【ここにCSVをドロップ】CSVをExcelで開く.bat  # 制御層
├── CsvToExcel_backend.ps1                        # 処理層
└── README.md                                     # 本ファイル
```

### バッチファイル仕様

| 項目 | 内容 |
|------|------|
| ファイル名 | `【ここにCSVをドロップ】CSVをExcelで開く.bat` |
| 責務 | 入力パラメータ取得、PowerShell実行、結果表示 |
| 入力チェック | ファイル未指定時はエラーメッセージを表示 |
| エラーハンドリング | PowerShellの終了コードを確認 |

### PowerShellスクリプト仕様

| 項目 | 内容 |
|------|------|
| ファイル名 | `CsvToExcel_backend.ps1` |
| 責務 | 文字コード判定、変換、Excel起動 |
| 使用技術 | COMオブジェクト、System.Windows.Forms |
| 一時ファイル | `%TEMP%` フォルダにGUID付きで作成 |

### システム構成図

```
[ユーザー操作]
       ↓
[.bat 制御層]
   - 入力チェック
   - PowerShell呼び出し
   - 結果表示
       ↓
[PowerShell 処理層]
   - 文字コード判定
   - UTF-8 BOM変換
   - 一時ファイル作成
       ↓
[Excel COM API]
       ↓
[Excelプロセス]
```

## 注意事項

### セキュリティについて

- 本ツールはローカルファイルのみ処理します
- インターネット通信は行いません
- 一時ファイルはユーザーの一時フォルダに作成されます

### 既知の制限事項

1. **Excel必須**: Excelがインストールされていない環境では動作しません
2. **大容量ファイル**: 非常に大きなCSV（100MB以上）の処理には時間がかかる場合があります
4. **特殊文字コード**: EUC-JP、ISO-2022-JPなどの古い文字コードは自動判定できません

### データの取り扱い

- 元のCSVファイルは変更されません
- 処理は読み取り専用で行われます
- 一時ファイルはユーザーの一時フォルダにのみ作成されます
