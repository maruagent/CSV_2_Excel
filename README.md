# CSV to Excel Opener (Drag & Drop)

## 1. 導入の目的と解決される課題

### 1.1 現状の問題点
Windows環境においてCSVファイルを扱う際、以下の仕様上の制約により、業務効率の低下とデータ破損のリスクが発生しています。

#### （１）標準のファイル関連付けによる実行
  * 現象：UTF-8形式（BOMなし）のCSVをダブルクリックで開くと、Excelが Shift_JIS と誤認して文字化けすることがある。（BOMは文字コードの識別用データ）
  * リスク：文字化けに気づかず上書き保存することで、元のデータが破損する。

#### （２）インポート機能の操作コスト
  * 現状：文字化けを回避するには、Excel起動後に「データ」タブ → 「テキストまたはCSVから」を選択し、文字コードを指定して読み込む必要がある。
  * 工数：ファイルを開くたびに5回以上のクリック操作と設定確認が要求される。

### 1.2 本ツールの解決策
本ツールは、CSVファイルを読み取り、文字コードを自動判定した上で、Excelが最も安全に読み取れる形式（UTF-8 BOM付き）へ変換し、一時ファイルを介してExcelを起動します。

#### （１）本ツールによる改善後
  * 操作：対象ファイルをバッチファイルへドラッグ＆ドロップするのみ。
  * 効果：文字コード判定からExcel起動までが自動化され、操作工数が約80%削減される。
  * 安全性：元のCSVファイルを直接書き換えず、一時ファイルとして開くため、オリジナルが維持される。

## 2. 動作原理と技術仕様

### 2.1 処理フロー
本ツールは制御層（Batch）と処理層（PowerShell）の二層構造で動作します。
```
[CSVファイル]
      ↓ (Drag & Drop)
[.bat 制御層]
      ↓ 引数(パス)をPowerShellへ渡す
[PowerShell 処理層]
      ├─ ① 文字コードの自動判定 (UTF-8 / UTF-16 / Shift-JIS)
      ├─ ② UTF-8 BOM (Byte Order Mark) への変換
      └─ ③ 一時ディレクトリ(%TEMP%)への出力 (GUID付与による衝突回避)
[Excel Application]
      └─ 作成された一時ファイルを展開
[バックグラウンド監視]
      └─ Excel終了を検知後、一時ファイルを物理削除
```
### 2.2 技術的優位性
  * **非破壊編集**: オリジナルのファイルには一切変更を加えない参照専用の起動。
  * **自動クリーンアップ**: `%TEMP%` 領域への書き出しと自動削除により、ストレージを圧迫しない。
  
## 3. 動作環境
#### （１）ソフトウェア要件
  * OS：Windows 10 / 11
  * アプリケーション：Microsoft Excel 2016 以降
  * ランタイム：Windows PowerShell 5.1（多くのWindowsにはインストールされてある）

#### （２）ハードウェア要件
  * 制限事項：100MBを超える大容量ファイルの場合、メモリ上のオブジェクト参照および変換処理に時間を要する場合がある。

## 4. 導入・使用方法
### 4.1 インストール
1.  リポジトリ内の `【ここにCSVをドロップ】CSVをExcelで開く.bat` と `CsvToExcel_backend.ps1` を同一フォルダに配置する。
2.  バッチファイルのショートカットをデスクトップ等、アクセスしやすい場所に作成する。

### 4.2 使用手順
1.  開きたいCSVファイル（複数選択可）を選択する。
2.  作成したショートカット、またはバッチファイル本体へドラッグ＆ドロップする。

## 5. トラブルシューティング
#### （１）スクリプト実行エラー
  * 原因：PowerShellの実行ポリシー（ExecutionPolicy）が制限されている。
  * 対策：本ツールはバッチファイル内で `-ExecutionPolicy Bypass` を指定しているため、通常は設定変更不要。
#### （２）特定の文字コードでの判定失敗
  * 原因：EUC-JP等の日本語レガシーエンコーディングは現在サポート外。
  * 対策：UTF-8またはShift-JISでの運用を推奨。

#### 【ファクトチェック結果】
